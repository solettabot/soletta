39 /*
40  * Data structure to hold callbacks to be later notified. It provides means to safely walk the
41  * callbacks, and defers deletion of elements until all the walks finish. For convenience, the
42  * SOL_MONITORS_WALK macro can be used to walk in the callbacks.
43  *
44  * A cleanup function can be provided to perform any extra cleanup for each element. This will be
45  * called by clear() function as well.
46  *
47  * By default it stores both a callback and a data pointer, this can be extended by embedding struct
48  * sol_monitors_entry in your own structure and use sol_monitors_init_custom() or
49  * SOL_MONITORS_INIT_CUSTOM macro.
50  */
51
